## 1. Introducción
[cite_start]**Objetivo:** Aplicar el uso de Git y GitHub para la gestión de versiones en proyectos de Ciencia de Datos y realizar preprocesamiento con Pandas[cite: 23, 24].
[cite_start]**Funcionalidades:** Se creó un script que elimina duplicados, gestiona nulos, codifica variables y normaliza datos numéricos[cite: 67].

## 2. Comandos Git usados
* `git clone`: Para copiar el proyecto a local.
* [cite_start]`git config`: Para establecer mi identidad de usuario[cite: 62].
* [cite_start]`git checkout -b`: Para crear la rama `feature-preprocesamiento`[cite: 66].
* [cite_start]`git add`: Para preparar el archivo `preprocesamiento.py`[cite: 68].
* [cite_start]`git commit`: Para guardar los cambios con un mensaje descriptivo[cite: 69].
* [cite_start]`git push`: Para subir la rama a la nube de GitHub[cite: 69].

## 3. Automatización (GitHub Actions)
Se implementó un flujo de trabajo que se activa con cada "push". [cite_start]El sistema crea un entorno virtual con Python, instala Pandas y Scikit-learn, y ejecuta el script para verificar que no existan errores lógicos o de sintaxis[cite: 82].
